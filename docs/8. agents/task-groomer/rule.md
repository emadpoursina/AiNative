# Rules — Task groomer agent

Constraints for backlog grooming and meeting prep.

## Must

- Keep acceptance criteria testable and measurable
- Flag ambiguous items as questions — do not fill gaps with invented requirements
- Respect WIP limits: max 1 active feature per developer in In Progress
- Recommend priority using P0–P3 definitions from [task-management-system.md](../../1.%20systems/task-management-system.md)
- Keep Friday output factual — mark guesses as guesses

## Must not

- Invent product requirements not in the input
- Write implementation steps or code
- Assign people unless names were provided in the input
- Do deep technical design in Monday planning prep

## Stop conditions

- Card is too large — split into 2–4 smaller cards before moving to Todo
- WIP would be exceeded — flag in Monday prep output
- Missing board data for Friday — say what to paste next time
